# Security Policy

Gracias por tu interés en la seguridad de Transmission Routes. Si encuentras una vulnerabilidad de seguridad, por favor repórtala de forma confidencial para que podamos atenderla y, si procede, corregirla.

## Cómo reportar
- Abre una nueva issue en el repositorio con el prefijo `[SECURITY]` en el título **y marca la issue como privada** si tu repositorio permite; o
- Envía un correo a `security@kevindels.dev` (reemplaza por tu contacto real) con:
  - Descripción clara del problema
  - Pasos para reproducirlo
  - Archivos de log o PoC (si aplicable)
  - Impacto estimado

## Proceso
- Confirmaremos recepción en 3 días hábiles y mantendremos comunicación por el canal que indiques.
- Intentaremos publicar un parche en un plazo razonable (normalmente 30–90 días, según gravedad).
- Si la vulnerabilidad afecta a usuarios finales de forma crítica, trabajaremos para coordinar notificaciones y parches.

## Política de divulgación
- Pedimos que no divulgues públicamente la vulnerabilidad hasta haber coordinado el parche o plan de mitigación.
- Consideraremos asignar un CVE cuando corresponda.

Gracias por ayudarnos a mantener este proyecto seguro.
